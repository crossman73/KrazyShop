# CosmicScan 사업 모델 재정의 - 이미지 기반 화장품 성분 분석 및 추천 서비스

## 🎯 서비스 핵심 목표 (사용자 요구사항 기반)

### 1차 목표: 이미지 기반 성분 분석
- **이미지 OCR**: 제품 포장지/용기에서 성분 정보 자동 추출
- **성분 유해성 분석**: EWG, 식약처 DB 기반 위험 성분 식별
- **성분 유익성 분석**: 보습, 미백, 안티에이징 등 효능 성분 분석
- **글로벌 인증 정보**: FDA, EMA, KFDA 승인 여부 확인

### 2차 목표: 지능형 제품 추천
- **유사제품 추천**: 동일/유사 기능 제품 추천
- **함께 사용하면 좋은 제품**: 시너지 효과 있는 제품 조합
- **같이 사용하면 안되는 제품**: 성분 충돌/과다 사용 위험 경고

### 3차 목표: 최저가 구매 지원
- **최저가 판매처 검색**: 14개 주요 쇼핑몰 실시간 가격 비교
- **가격 알림**: 목표 가격 도달 시 알림 서비스
- **쿠폰/할인 정보**: 추가 할인 혜택 정보 제공

## 📊 시장 조사 및 기술 동향

### 화장품 안전성 평가 제도 시행 (2025년 현재)
- **2028년 단계적 시행**: 화장품 안전성 평가제도 의무화 예정
- **성분별 독성 데이터**: 원료부터 완제품까지 안전성 입증 필요
- **소비자 안전 의식 증가**: 성분 투명성 요구 급증

### 개인화 추천 시스템 기술 발전
- **하이브리드 AI 추천**: 협업필터링 + 콘텐츠 기반 필터링 조합
- **실시간 개인화**: 사용자 행동 기반 즉시 추천
- **설명 가능한 AI**: 추천 이유 명확히 제시하는 트렌드

### 가격 비교 서비스 성숙도
- **실시간 크롤링**: 주요 쇼핑몰 가격 정보 자동 수집
- **신뢰도 검증**: 정확한 가격 정보 제공 시스템 구축
- **법적 준수**: 크롤링 규정 준수 및 공정거래 원칙

## 🔧 핵심 기능 상세 설계

### 1단계 MVP 기능 (필수도 9-10점)

#### 이미지 OCR 제품 인식 (개발 5일)
```yaml
기술스택:
  - Primary: Tesseract.js (클라이언트 OCR)
  - Backup: Google Vision API (월 1000건)
  - 전처리: OpenCV.js (이미지 최적화)

인식대상:
  - 제품명: 브랜드 + 제품명 추출
  - 성분표: 전성분 리스트 인식
  - 용량정보: ml, g 등 용량 추출
  - 바코드: 제품 식별용 백업

정확도목표: 95% 이상 (한글/영문)
```

#### 성분 유해성 분석 (개발 7일)
```yaml
데이터소스:
  - EWG Skin Deep Database: 글로벌 성분 안전성 DB
  - 식약처 화장품 성분 사전: 국내 규제 정보
  - INCI Dictionary: 국제 성분 명명법
  - EU SCCS 의견서: 유럽 안전성 평가

위험등급:
  - 0-2점: 안전 (초록)
  - 3-6점: 보통 위험 (노랑)  
  - 7-10점: 높은 위험 (빨강)

주요위험성분:
  - 발암물질: IARC 분류 기준
  - 환경호르몬: 내분비계 교란물질
  - 알레르기 유발: 26가지 향료 알레르겐
  - 자극성분: 민감성 피부 주의 성분
```

#### 성분 유익성 분석 (개발 5일)  
```yaml
효능카테고리:
  - 보습성분: 히알루론산, 글리세린, 세라마이드
  - 안티에이징: 레티놀, 비타민C, 펩타이드
  - 미백성분: 나이아신아마이드, 알부틴, 비타민C
  - 진정성분: 알로에, 센텔라, 판테놀

효과등급:
  - A등급: 입증된 효과 (임상연구 기반)
  - B등급: 일반적 효과 (사용자 후기 기반)
  - C등급: 미미한 효과 (이론적 근거)

농도정보: 효과 발현 최소 농도 표시
```

### 2단계 고급 기능

#### 유사제품 추천 시스템 (개발 6일)
```yaml
추천알고리즘:
  - 성분 유사도: 주성분 80% 이상 일치
  - 기능 유사도: 동일 카테고리 + 효능 매칭
  - 가격대 고려: ±30% 범위 내
  - 브랜드 다양성: 동일 브랜드 최대 2개

추천개수: 5-8개 제품
필터링: 피부타입, 예산, 브랜드 선호도
```

#### 제품 조합 추천/경고 (개발 8일)
```yaml
함께_사용_좋은_조합:
  - 비타민C + 비타민E: 항산화 시너지
  - 히알루론산 + 세라마이드: 보습 강화
  - 나이아신아마이드 + 아연: 트러블 케어

사용_금지_조합:
  - AHA/BHA + 레티놀: 과도한 자극
  - 비타민C + 레티놀: 효과 상쇄
  - 벤조일퍼옥사이드 + 레티놀: 자극 위험

경고레벨:
  - 주의: 사용간격 조절 권장
  - 위험: 동시 사용 금지
  - 심각: 피부 손상 위험
```

## 🏗️ 기술 아키텍처 설계

### POC 단계 무료 환경 구성

#### Frontend 구조
```yaml
사용자앱 (app.cosmicscan.com):
  - 이미지 업로드 + OCR
  - 성분 분석 결과 표시
  - 제품 추천 목록
  - 가격 비교 화면

기술스택:
  - Next.js 14 + TypeScript
  - TailwindCSS + Headless UI
  - PWA 지원 (모바일 최적화)
  - 카메라 API 통합
```

#### Backend 마이크로서비스
```yaml
Supabase Edge Functions:
  - analysis-service: OCR + 성분 분석
  - recommendation-service: 제품 추천 로직
  - price-service: 가격 크롤링 + 비교
  - product-service: 제품 DB 관리

Database Schema:
  - products: 제품 기본 정보
  - ingredients: 성분 마스터 DB
  - safety_data: 안전성 평가 데이터
  - price_history: 가격 변동 이력
  - user_profiles: 사용자 선호도
```

#### AI/ML 서비스 통합
```yaml
OCR Engine:
  - Tesseract.js: 무료 클라이언트 OCR
  - Google Vision API: 백업 (1000건/월)

추천 Engine:
  - 콘텐츠 기반 필터링: 성분 유사도
  - 협업 필터링: 사용자 행동 패턴
  - 하이브리드 가중평균: 70% + 30%

자연어처리:
  - 성분명 표준화: 영문↔한글 매핑
  - 리뷰 감성분석: 긍정/부정 분류
  - 키워드 추출: 주요 효능 단어
```

### 프로덕션 확장 아키텍처

#### 마이크로서비스 분리
```yaml
Core Services:
  - user-service: 회원 관리 + 프로필
  - product-service: 제품 DB + 크롤링
  - analysis-service: 성분 분석 엔진
  - recommendation-service: AI 추천
  - price-service: 가격 비교 + 알림
  - community-service: 리뷰 + 평점

Data Services:
  - ingredient-db: 성분 마스터 데이터
  - safety-db: 안전성 평가 DB
  - price-db: 가격 이력 데이터
  - analytics-db: 사용자 행동 분석

Infrastructure:
  - API Gateway: Kong 또는 AWS API Gateway
  - Message Queue: Redis Pub/Sub
  - Cache Layer: Redis Cluster
  - CDN: CloudFlare
  - Monitoring: DataDog + Sentry
```
